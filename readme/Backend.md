### 💡 Чего не хватает (Архитектурно и согласно изначальному плану)


* [ ]  **API для заявок в друзья:** Маршрут `POST /api/friend` сейчас сразу присваивает связи статус `accepted`. Не хватает эндпоинта для подтверждения/отклонения входящих заявок (например, `PUT /api/friend/accept`).


* [ ] **API для лайков:** Эндпоинты `POST /api/post/like` и `DELETE /api/post/like` для добавления функционала, спроектированного нами на предыдущем этапе.

### 🚨 Чего не хватает (Разрыв между фронтендом и бэкендом)


В клиентском JavaScript-коде (Vue) прописаны запросы к эндпоинтам, которых **ещё не существует** на сервере Go. Из-за этого часть интерфейса сейчас выдает ошибки при использовании:

* [ ] **`GET /getUserId`** — Скрипт чата (`chat.js`) пытается получить ID текущего авторизованного пользователя для логики сообщений, но сервер не обрабатывает этот URL.


* [ ] **`/ws`** — Отсутствует обработчик WebSocket-соединений, необходимый для работы чата в реальном времени.


* [ ] **`GET /api/AllMessage`** — Бэкенд не умеет отдавать историю переписки между двумя пользователями.


* [ ] **`PUT /api/message`** и **`DELETE /api/message`** — Сервер не имеет маршрутов для редактирования и удаления сообщений из базы данных.


* [ ] **`DELETE /api/friend`** — Во фронтенде (`profile.js`) заложена функция удаления из друзей `deleteFriend`, но серверный `FriendHandler` обрабатывает только методы `GET` и `POST`.


### 📄 Существующие страницы (HTML-шаблоны)

Эти страницы сервер отдает пользователю при переходе по соответствующим URL:

* **`/login`** — Страница авторизации (`login.html`).


* **`/registration`** — Страница регистрации (`registration.html`).


* **`/profile`** — Личный кабинет пользователя (`profile.html`).


* **`/friend`** — Просмотр чужого профиля (`friend.html`).


* **`/chat`** — Окно переписки с конкретным пользователем (`chat.html`).



### 🔌 Существующие API-эндпоинты (Бэкенд)

Эти маршруты обрабатывают бизнес-логику и возвращают данные:

* **Аутентификация:**
* `POST /login` — Проверка данных и создание сессии.


* `POST /registration` — Создание нового пользователя.


* `GET /exit` — Уничтожение сессии (выход из аккаунта).




* **Профиль пользователя:**
* `GET /api/profile` — Получение данных своего профиля (или чужого, если передан параметр `id`).


* `PUT /api/profile` — Обновление имени и информации «о себе».


* `POST /api/profile/avatar` — Загрузка фотографии пользователя.




* **Стена (Посты):**
* `GET /api/post` — Получение списка постов.


* `POST /api/post` — Создание нового поста (включая загрузку картинки).


* `PUT /api/post` — Редактирование текста и заголовка поста.


* `DELETE /api/post` — Удаление поста.




* **Друзья:**
* `GET /api/friend` — Получение списка друзей или поиск пользователей по имени (параметр `name`).


* `POST /api/friend` — Добавление пользователя в друзья.
